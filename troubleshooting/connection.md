# Проблемы подключения к АТС

## Сервис не подключается к AMI

**Симптомы:** В личном кабинете статус подключения — «Ошибка» или «Не подключено».

**Возможные причины и решения:**

1. **Порт 5038 закрыт**
   - Проверьте, что порт 5038 TCP открыт на файрволе
   - Для Yeastar: убедитесь, что порт проброшен через NAT
   ```bash
   # Проверка доступности порта с внешнего сервера
   telnet ВАШ_IP 5038
   ```

2. **Неверные учётные данные AMI**
   - Проверьте логин и пароль в настройках сервиса
   - Для FreePBX: убедитесь, что данные совпадают с `/etc/asterisk/manager_custom.conf`

3. **IP-адреса Callbee не разрешены**
   - Для FreePBX: проверьте `permit` в `manager_custom.conf`
   - Для Yeastar: проверьте «Разрешённые IP/Маска» в настройках AMI
   - Актуальные IP-адреса: [IP-адреса сервиса](/ip-addresses/)

4. **bindaddr не изменён** (FreePBX)
   - В `/etc/asterisk/manager.conf` значение `bindaddr` должно быть `0.0.0.0`
   - После изменения выполните: `asterisk -rx "manager reload"`

## API Yeastar недоступен

**Для моделей S50, S100, S300:**

1. Убедитесь, что API активирован в настройках АТС
2. Проверьте доступность порта 8088 TCP
3. Проверьте протокол (HTTP/HTTPS)
