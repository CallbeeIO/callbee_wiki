# Проблемы с записями разговоров

## Записи не появляются в CRM

**Для FreePBX:**

1. Проверьте, что контейнер nginx запущен:
   ```bash
   docker ps | grep recs_nginx
   ```
2. Проверьте доступность записей по HTTP:
   ```bash
   curl http://ВАШ_IP:50381/monitor/
   ```
3. Убедитесь, что порт 50381 TCP открыт для CRM-портала
4. Для amoCRM: требуется **HTTPS** с валидным SSL-сертификатом

**Для Yeastar S50/S100/S300:**

1. Проверьте, что API активирован и доступен
2. Убедитесь, что порт 8088 проброшен через NAT

**Для Yeastar S20:**

1. Проверьте, что FTP активирован
2. Убедитесь, что порт 21 проброшен через NAT
3. Проверьте учётные данные FTP (логин: `support`)

## Записи обрезаны или плохого качества

Это проблема на стороне АТС, а не Callbee. Проверьте настройки записи в вашей АТС.
